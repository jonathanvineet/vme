## Methodology

- **Ground truth is engineer-authored, never generated.** The loader enforces this mechanically: `metadata.json` must name a real `labeled_by`; values like `auto`/`generated`/`pipeline` are rejected with `GroundTruthError`. The benchmark never writes into `ground_truth/`.
- **Selectors, not UUIDs.** An engineer labels a physical bar as a list of `{drawing, mark}` pairs — things a human can read off a sheet — and the loader resolves them to pipeline observation UUIDs. Every resolution failure carries its reason (`drawing_missing` / `mark_missing` / `index_out_of_range`), so recall loss is always attributable to a pipeline stage, not just a number.
- **Strict matching (v1).** An accepted identity is *correct* only if its observation set exactly equals a ground-truth identity's resolved set. Partial overlap is classified as a merge/split/partial error, never partial credit — partial credit would hide exactly the failures the corpus exists to expose.
- **Undefined metrics stay undefined.** Precision with zero accepted identities is `null` with the note "undefined: zero identities were accepted", not a fabricated 0 or 1 — the same absence-is-a-fact discipline as everywhere else in this project.
- **Metric definitions**: precision = exactly-correct accepted / accepted; recall = exactly-recovered GT / GT total; false merge = accepted identity spanning ≥2 GT bars; false split = one GT bar scattered across ≥2 accepted identities; observation coverage = GT selectors resolved / total; engineering coverage = GT-asserted facts (mark/diameter/spacing/shape) recoverable from resolved observations / total asserted; reconstruction coverage = GT bar marks with ≥1 recovered physical bar (existence only, never geometry quality).
- **Explainability is structural, not decorative.** Every GT outcome and every pipeline-identity outcome carries a `status` + `reason` + the specific selector failures behind it; `summary.md` and `report.html` render these verbatim, and the error gallery draws each failure's observation bboxes grouped by GT-vs-pipeline ownership.
- **Determinism by construction**: no timestamps anywhere in report content, all collections sorted, `sort_keys=True` — two runs produce byte-identical `report.json` (regression-tested).

## Known limitations

- **The corpus contains zero real projects.** The fixture's labels are synthetic (its own `metadata.json` says so) and exist to test metric math, not engineering truth. Every metric this framework reports is meaningless until an engineer labels real packages — the framework is the measuring instrument, built ahead of the thing it measures, deliberately.
- Exact-match correctness (v1) will read legitimate near-misses harshly once real data arrives; if that becomes noise rather than signal, add a separately-reported relaxed tier — never blend it into the strict number.
- Reconstruction coverage is existence-by-mark only; geometric-fidelity ground truth (centerline coordinates) is a corpus-format extension for later, not silently approximated now.
- The error-gallery renders observation bboxes in each drawing's own coordinates; observations from different drawings share no frame (engineering_assumptions.md A12), so multi-drawing error images juxtapose rather than align — correct, but worth knowing when reading them.
