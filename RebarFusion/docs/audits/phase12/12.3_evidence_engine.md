# Phase 12.3 ‚Äî Evidence Engine

**Scope, as specified before this subphase started: transform Phase 12.2's qualitative evidence into structured, explainable scores. No accept/reject/review decisions. No merging. No geometry. Reuse Phase 9.4's confidence-decomposition philosophy ‚Äî a per-category breakdown, geometric mean, always-traceable weakest category ‚Äî rather than one opaque number.**

## What was built

- `core/fusion/models.py` ‚Äî `EvidenceScore` (`category_scores`, `overall`, `weakest_category`, `unscored_categories`), `ScoredCandidate` (one anchor/candidate pair, `is_candidate` carried over unchanged from 12.2, `evidence`, `score`), `ScoredHypothesis` (`status` always `PENDING`). Also: `EVIDENCE_CATEGORY_*`/`POLARITY_*` string constants, and `HYPOTHESIS_REVIEW` ‚Äî reserved for Phase 12.4, not used by 12.2 or 12.3.
- `core/fusion/evidence_engine.py` ‚Äî `score_hypotheses(hypotheses, observations)`.
- `run_phase12.py` extended to write `debug/<project>/scored_hypotheses.json`.
- `tests/test_phase12_evidence_engine.py` ‚Äî 5 checks. All pass.

## The `HypothesisEvidence` -> `Evidence` extension (not a 12.2 reopening)

Per review, `HypothesisEvidence` gained `uuid`, `category`, `polarity`, and `confidence: Optional[float] = None`, and was renamed `Evidence` ‚Äî a shared type both 12.2 and 12.3 now use, rather than 12.3 wrapping 12.2's type in something new. This is the same pattern as `ObservationFact` gaining `source_entity_uuid` during 12.2: **12.2's generation logic, qualification rules, and real-data results are unchanged** ‚Äî verified directly, not just asserted: `tests/test_phase12_hypothesis_generator.py` still asserts the exact same `N4->{}`, `N6->{N7}`, `N7->{N6}` real-data result after the rename, and still passes. What changed is that each `Evidence` item now additionally carries a category/polarity label (assigned by 12.2's existing rules, mechanically, not new decisions ‚Äî `same_mark` was always support-for, `fact_conflict` was always evidence-against, the rename just makes that explicit) and a `confidence` slot that 12.2 always leaves `None` (enforced by `check_no_decisions`, which now checks real data instead of just the dataclass shape, since the field's mere existence is correct and expected).

## Evidence category vocabulary ‚Äî only what has a real generator

`EVIDENCE_CATEGORY_*` names 8 categories (`DRAWING_CONTEXT`, `ENGINEERING_CONTEXT`, `ANNOTATION`, `ROLE`, `FACT`, `SPATIAL`, `TOPOLOGY`, `SCHEDULE`), matching the vocabulary from review. Only `ENGINEERING_CONTEXT` (`same_mark`), `ROLE` (`complementary_role`), and `FACT` (`fact_agreement`/`fact_conflict`) are ever scored ‚Äî `SPATIAL` evidence exists (every pair with position data gets a `spatial_distance` entry) but is deliberately excluded from `EvidenceScore.category_scores`, for a specific, evidence-grounded reason: **`spatial_distance`'s polarity is `UNKNOWN`, not `SUPPORTS`/`CONTRADICTS`, because no calibrated proximity threshold exists yet** (this project has exactly one real drawing's worth of cross-object distance data). Assigning a hard "close enough" cutoff to convert distance into supports/contradicts would be precisely the kind of hardcoded-threshold shortcut this project has avoided everywhere else (Phase 7.6's plausibility rule used a modified z-score instead of a fixed length cutoff, for the same reason). `DRAWING_CONTEXT`/`ANNOTATION`/`TOPOLOGY`/`SCHEDULE` are named but have zero generators today ‚Äî same discipline as `ASPECT_*` omitting `HOOK`/`LEVEL` in Phase 12.1: the vocabulary is complete and stable, but nothing is fabricated to fill categories that have no real source.

## Confidence provenance ‚Äî reused, not invented

Every populated `Evidence.confidence` is a geometric mean of two real numbers an earlier phase already computed:
- `same_mark`: geometric mean of the two `MARK` facts' own `confidence` (Phase 12.1, itself `family.confidence` from Phase 9).
- `complementary_role`: geometric mean of the two `DrawingRole.confidence` values (Phase 12.1's `classify_drawing_role`).
- `fact_agreement`/`fact_conflict`: geometric mean of the two contributing facts' confidences for whichever aspect the evidence is about.
- `spatial_distance`: stays `None` ‚Äî no real source to reuse, so nothing is assigned (see above).

No new confidence value is invented anywhere in this subphase. Verified directly (`check_confidence_provenance`): every non-spatial evidence item scored during this run carries a real `(0, 1]` confidence, every spatial one stays `None`.

## `EvidenceScore` computation

Per candidate pair: for each scorable category, `score = supports_confidence_sum / (supports_confidence_sum + contradicts_confidence_sum)` among that category's evidence (confidence-weighted, not a bare count ‚Äî this is the one place this subphase goes beyond the literal Phase 9.4 pattern, since Phase 9.4's dimensions don't have this supports/contradicts structure to weight). `overall` is the geometric mean across categories that actually have evidence (`unscored_categories` lists the rest explicitly, never silently omitted). `weakest_category` is always traceable, matching Phase 9.4.

## Real-data result

The `N6`<->`N7` candidate pair (Phase 12.2's only real candidate pairing) scores `category_scores={'fact': 0.75}`, `overall=0.75`, `unscored_categories=['engineering_context', 'role']`. This is not a fabricated-looking round number: 3 `fact_agreement` items (orientation/quantity/shape, each confidence 1.0 ‚Äî directly measured geometry) plus 1 `fact_conflict` (length, also confidence 1.0) give `3.0 / (3.0 + 1.0) = 0.75` exactly. `engineering_context`/`role` are correctly `unscored` ‚Äî no `same_mark` evidence exists (the marks genuinely differ) and no `complementary_role` evidence exists (both observations are `mould_instance`, not a complementary pair) ‚Äî so the engine honestly reports "no data in these categories" rather than defaulting them to 0 or 1. `N4`, which Phase 12.2 never qualified as anyone's candidate, correctly never appears in any `scored_candidates` list at all ‚Äî Phase 12.3 quantifies 12.2's candidates, it does not re-open who counts as one.
